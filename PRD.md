# Graph Chat - 产品需求文档

## 1. 产品概述

### 1.1 产品定位
**Graph Chat** 是一款基于图结构的协同研究 Agent，通过非线性对话方式帮助用户深度探索复杂话题。

核心差异化：
- 传统对话：线性时间轴，上下文容易混乱
- Graph Chat：树状/网状结构，用户可自由选择深入方向

### 1.2 一句话描述
> "像浏览网页一样探索对话——点击感兴趣的内容深入，Agent 辅助标注推荐方向，人机协同完成深度研究。"

### 1.3 目标用户
- 需要深度调研的学习者、研究者
- 做决策前需要多维度对比的职场人士
- 希望系统学习某个领域的自学者

## 2. 核心功能

### 2.1 智能标注
| 功能 | 说明 |
|------|------|
| 自动标注 | LLM 回答时自动识别可深入的概念/主题 |
| 显示形式 | 虚线下划线，hover 高亮 |
| 点击交互 | 点击后弹出推荐深入方向列表 |
| 开关控制 | 用户可开启/关闭标注显示 |
| 设计原则 | 轻量无侵入，不影响正常阅读 |

### 2.2 手动分叉
| 功能 | 说明 |
|------|------|
| 触发方式 | 选中 AI 回答中的任意文字 |
| 交互反馈 | 选中后显示"🔍 深入讨论"按钮 |
| 行为 | 点击后以选中文本为起点创建子对话 |

### 2.3 Agent 建议式自主分叉
| 功能 | 说明 |
|------|------|
| 触发条件 | AI 检测到回答包含多个独立维度 |
| 建议形式 | 输入框上方显示建议条 |
| 用户设置 | 可调整并行数（2-5）、探索深度（2-5轮） |
| 用户选择 | 确认创建 / 忽略 / 手动选择 |
| 确认后行为 | Agent 自动在各分支并行探索 |

### 2.4 自主探索
| 功能 | 说明 |
|------|------|
| 探索方式 | AI 在分支内自问自答推进 |
| 深度控制 | 达到设定轮数或目标达成后停止 |
| 状态显示 | 树状导航显示 🔄 探索中 / ✓ 完成 |
| 用户介入 | 随时可暂停、修改方向、终止 |
| 介入后 | AI 仍可在后续建议继续自动探索 |

### 2.5 综合结论
| 功能 | 说明 |
|------|------|
| 显示条件 | 仅在有后代分支的非主对话显示按钮 |
| 合并目标 | 用户自选任意祖先节点 |
| 参与分支 | 用户自选后代节点，可多选，不必同辈 |
| 源分支处理 | 保留 / 归档 / 删除（用户自选） |
| 结论形式 | 作为目标节点的系统上下文消息 |
| 多次合并 | 支持 |

## 3. 交互流程

```
用户提问
    ↓
LLM 回答 + 自动标注可深入点
    ↓
Agent 判断：是否建议分叉？
    ├── 否 → 正常对话，用户可点击标注或手动选中分叉
    │
    └── 是 → 显示建议条（并行数/深度可调）
              ↓
        用户 [确认创建并自主探索]
              ↓
        创建多个分支，AI 自动开始探索
              ↓
        各分支并行推进（显示进度 🔄）
              ↓
        Agent 判断：是否综合结论？
              ↓
        用户选择合并目标 + 参与分支 + 处理方式
              ↓
        综合结论插入目标节点上下文
              ↓
        用户 [满意] / [某分支再深入] / [重新探索]
```

## 4. 技术架构

### 4.1 前端
| 技术 | 选型 |
|------|------|
| 框架 | React + TypeScript |
| 构建工具 | Vite |
| 样式 | Tailwind CSS |
| 状态管理 | Zustand |
| 图标 | Lucide React |

### 4.2 后端
| 技术 | 选型 |
|------|------|
| 框架 | Python + FastAPI |
| 数据库 | PostgreSQL |
| ORM | SQLAlchemy |
| LLM | OpenAI API（可扩展） |

### 4.3 数据模型

```python
# 消息节点（关系通过 message_relations 表存储）
class Message:
    id: UUID
    conversation_id: UUID
    role: str           # user / assistant / system
    content: str
    node_type: str      # normal / annotation / merge
    created_at: datetime

# 对话分支
class Conversation:
    id: UUID
    parent_id: UUID     # null = 主对话
    name: str
    status: str         # active / exploring / done / archived
    fork_from: UUID
    fork_text: str      # 分叉时选中的文本
    auto_exploring: bool

# 合并记录
class MergeRecord:
    id: UUID
    target_id: UUID     # 合并到哪个节点
    source_ids: List[UUID]
    conclusion: str
    keep_option: str     # keep / archive / delete
    created_at: datetime
```

## 5. 界面设计

### 5.1 布局
```
┌─────────────────────────────────────┬──────────────┐
│  Header (Logo + 面包屑 + 智能标注开关) │  对话树       │
├─────────────────────────────────────┤  ───────────  │
│  Agent 建议条（条件显示）              │  主对话       │
├─────────────────────────────────────┤  ↳ 分支A      │
│  进度指示器（条件显示）                │    ↳ 分支A1  │
├─────────────────────────────────────┤  ↳ 分支B      │
│                                     │              │
│  消息列表                            │  [综合结论]   │
│  ┌─────────────────────────────┐  │  （条件显示） │
│  │ 用户消息                      │  │              │
│  └─────────────────────────────┘  │              │
│  ┌─────────────────────────────┐  │              │
│  │ AI 消息（含标注下划线）       │  │              │
│  └─────────────────────────────┘  │              │
│                                     │              │
├─────────────────────────────────────┴──────────────┤
│  输入框                                            │
└────────────────────────────────────────────────────┘
```

### 5.2 关键交互细节
- **标注**：默认显示虚线下划线，hover 变紫色
- **建议条**：渐变背景，图标 + 文字 + 设置 + 按钮分层
- **进度条**：动画圆点 + 文字状态
- **树状导航**：当前节点高亮，状态图标右对齐
- **合并弹窗**：单选目标 + 多选分支 + 下拉处理选项

## 6. 产品路线图

### Phase 1: MVP（当前）
- [ ] 基础对话 + 智能标注
- [ ] 手动分叉 + 树状导航
- [ ] Agent 建议式自主分叉
- [ ] 自主探索 + 状态显示
- [ ] 灵活合并综合结论

### Phase 2: 增强
- [ ] 接入真实 LLM API
- [ ] 数据持久化（PostgreSQL）
- [ ] 用户偏好学习
- [ ] 标注智能化（动态生成）

### Phase 3: Agent 化
- [ ] 目标导向的探索规划
- [ ] 工具调用（搜索、代码执行）
- [ ] 跨分支记忆共享
- [ ] 多用户协作

## 7. 竞品差异

| 产品 | 特点 | Graph Chat 差异 |
|------|------|----------------|
