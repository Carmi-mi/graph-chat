# Graph Chat - 交互设计文档

## 1. 智能标注交互

### 1.1 默认状态
- AI 回答中的可深入点显示为 **虚线下划线**（`border-bottom: 1px dotted #999`）
- 不影响正常阅读，视觉上轻微区分

### 1.2 Hover 状态
- 下划线变紫色（`#667eea`）
- 背景轻微高亮（`rgba(102, 126, 234, 0.08)`）
- 文字颜色变紫色

### 1.3 点击状态
- 下划线加粗（`2px solid #667eea`）
- 背景更明显（`rgba(102, 126, 234, 0.12)`）
- 弹出推荐浮层

### 1.4 推荐浮层
- 位置：标注文字下方，带小三角箭头
- 内容："💡 推荐深入" + 推荐列表
- 列表项：hover 变紫色背景
- 点击列表项：创建分支并进入

### 1.5 关闭
- 点击浮层外任意位置关闭
- 移除 active 状态

## 2. 手动分叉交互

### 2.1 选中文本
- 用户在 AI 回答中拖动选中任意文字
- 文字长度限制：1-100 字符

### 2.2 显示 Tooltip
- 位置：选中文字上方居中
- 样式：黑色圆角按钮，"🔍 深入讨论"
- 动画：缩放淡入

### 2.3 点击 Tooltip
- 创建子对话分支
- 清除选中状态
- 自动切换到新分支

### 2.4 取消
- 点击 tooltip 外任意位置
- 或继续输入新消息

## 3. Agent 建议条交互

### 3.1 触发条件
- AI 回答包含 3+ 个标注点
- 或检测到"对比""选择""哪个"等关键词

### 3.2 显示位置
- 输入框上方
- 渐变背景（紫色到白色）

### 3.3 布局结构
```
┌─────────────────────────────────────────┐
│  [💡]  Agent 建议：检测到 4 个独立维度... │
│                                         │
│  并行数：[3 ▼]  深度：[3轮 ▼]            │
│                                         │
│       [忽略]  [创建并自主探索]           │
└─────────────────────────────────────────┘
```

### 3.4 用户操作
| 操作 | 结果 |
|------|------|
| 调整并行数 | 下拉选择 2-5 |
| 调整深度 | 下拉选择 2-5 轮 |
| 点击忽略 | 建议条消失，恢复正常对话 |
| 点击创建 | 创建分支，开始自主探索 |

## 4. 自主探索交互

### 4.1 探索中状态
- 进度指示器显示在输入框上方
- 动画圆点 + "Agent 正在 X 个分支并行探索..."
- 树状导航对应分支显示 🔄

### 4.2 完成状态
- 进度指示器变为 ✓ + 绿色文字
- 树状导航对应分支显示 ✓
- 提示"可综合结论"

### 4.3 用户介入
- 点击分支节点可查看实时内容
- 在分支内输入消息：**立即终止**该分支的自动探索，状态变为 `active`
- 切换到其他分支：探索继续，不暂停
- 关闭页面：探索任务终止，不持久化（MVP 阶段）

### 4.4 探索状态机
```
exploring ── 用户输入消息 ──→ active（立即终止探索，回到普通对话）
exploring ── 达到 max_depth ──→ done
exploring ── 检测到收敛 ──→ done
done ── 用户点击综合结论 ──→ 进入合并流程
```

MVP 阶段不做：暂停/恢复、后台持久化、WebSocket 实时推送。探索进度通过轮询 GET /api/agent/status/{convId} 获取。

## 5. 综合结论交互

### 5.1 按钮显示条件
- 当前不是主对话
- 当前分支有后代分支

### 5.2 点击按钮
- 弹出模态框
- 遮罩层 + 居中卡片

### 5.3 模态框内容
```
┌─────────────────────────────┐
│  📋 综合结论                  │
│  选择合并目标和参与分支        │
├─────────────────────────────┤
│  合并到                       │
│  ○ 主对话                     │
│  ○ 分支A（当前）              │
├─────────────────────────────┤
│  参与分支（可多选）            │
│  ☑ 分支A1  来自：分支A /      │
│  ☐ 分支A2  来自：分支A /      │
│  ☑ 分支B    来自：主对话 /    │
├─────────────────────────────┤
│  源分支：[保留 ▼]  [取消] [确认]│
└─────────────────────────────┘
```

### 5.4 选择交互
| 元素 | 交互 |
|------|------|
| 目标选项 | 点击整行选中，radio 同步，紫色边框 |
| 分支选项 | 点击整行切换 checkbox，选中后紫色边框+背景 |
| 源分支下拉 | 保留/归档/删除 |

### 5.5 确认后
- 模态框关闭
- 目标对话插入系统消息（综合结论）
- 源分支按选择处理（保留/归档/删除）
- 树状导航刷新

---

## 6. 树状导航交互

### 6.1 节点状态
| 状态 | 显示 |
|------|------|
| 当前选中 | 紫色背景，文字加粗 |
| 探索中 | 节点名右侧 🔄 |
| 完成 | 节点名右侧 ✓ |
| 普通 | 无特殊标记 |

### 6.2 点击节点
- 切换到对应分支对话
- 消息列表刷新
- 面包屑更新
- 综合结论按钮状态更新

### 6.3 层级缩进
- 每级缩进 16px
- 左侧竖线连接父子

---

## 7. 面包屑交互

### 7.1 显示格式
`主对话 / 分支A / 分支A1`

### 7.2 更新时机
- 切换分支时
- 创建新分支时
